# Implementation Plan: Windows Support via WSL2 Backend

**Branch**: `001-windows-wsl-support` | **Date**: 2026-02-25 | **Spec**: [spec.md](spec.md)

## Summary

Enable Parallel Code to run as a native Windows desktop application by delegating all POSIX-dependent operations (PTY, shell, git, symlinks) to a WSL2 distro. The Electron main process detects WSL2 at startup, translates Windows paths to WSL mount paths, and routes `pty.spawn()` and `exec('git', …)` calls through `wsl.exe`. No new IPC channels are required; all changes are encapsulated in existing handler modules. A NSIS installer is added to the electron-builder config for x64 and arm64 Windows targets.

## Technical Context

**Language/Version**: TypeScript (strict), Node.js 18+  
**Primary Dependencies**: Electron, node-pty (Win32 ConPTY prebuilds included), SolidJS  
**Storage**: N/A — no new persistent data; runtime WSL state cached in `process.env`  
**Testing**: Vitest (existing unit tests must continue to pass); manual E2E on Windows 10 21H2+ and Windows 11  
**Target Platform**: Windows 10 21H2+ with WSL2; extends existing macOS 12+ and Linux (Ubuntu/Debian)  
**Project Type**: Desktop app (Electron, two-process renderer + main)  
**Performance Goals**: PTY sessions spawn within 3 seconds; WSL2 detection completes within 5 seconds at startup  
**Constraints**: Zero regressions on macOS/Linux; lint budget stays ≤ 21 warnings; strict TypeScript throughout; all existing tests green  
**Scale/Scope**: 6 existing files modified; 1 new icon asset; ~200–300 lines of new platform-guarded code
